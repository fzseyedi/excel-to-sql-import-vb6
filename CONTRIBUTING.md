# Contributing

Thank you for your interest in contributing to this project.

## Ways to contribute

You can help by:

- reporting bugs
- suggesting improvements
- improving documentation
- refactoring code
- adding validation and usability improvements
- testing with real-world Excel/SQL Server scenarios

## Before opening a pull request

Please make sure that:

- the change is clear and focused
- the code is consistent with the current project style
- existing functionality is not broken
- changes are tested as much as possible

## Bug reports

When reporting a bug, please include:

- what you expected to happen
- what actually happened
- steps to reproduce the problem
- screenshots if the issue is UI-related
- sample error message or log output if available

## Feature requests

Feature requests are welcome.

Please try to explain:

- the problem you are trying to solve
- why the feature would be useful
- a simple example of the expected behavior

## Pull requests

Small, focused pull requests are preferred.

Please avoid mixing unrelated changes in one pull request.

## Code style

This project is built in VB6, so contributions should aim to keep the code:

- readable
- simple
- consistent with the current structure
- easy to debug and maintain

## Notes

Some parts of the application depend on local environment setup, including:

- VB6 runtime
- SQL Server access
- ADO / MDAC
- optional Excel Automation support

Because of that, environment-related notes in issues and pull requests are very helpful.
